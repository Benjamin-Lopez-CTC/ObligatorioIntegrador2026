# Prompt 30 (B)

Vamos a hacer unos cambios a las colmenas, ahora las notas técnicas pasarán a ser mucho mas importantes, registrando desde ahí la temperatura, la humedad, y la cantidad de alzas, medias alzas o alzas 3/4 de miel cosechadas, ademas de los datos que ya se guardan en las notas.

También, cuando se crea una colmena, se debe poder ingresar la temperatura, humedad y la cantidad de alzas de cada tipo que tendrá la colmena, con un selector de añadir y especificar, donde se muestran los tres tipos de alzas y se selecciona la cantidad por cada una, siguiendo limites logicos de espacio que caben sobre una colmena, los kilos de miel registrados se deben auto-ingresar.

Se tendrá que añadir otra vista, la cual permitirá hacer extracciones en masa, junto con un registro histórico de dichas extracciones.
Esta vista tendrá el registro de extracciones y un botón para realizar una cosecha, la cual al apretarse abrirá un modal que permita añadir las colmenas a cosechar, filtradas y separadas por apiario con un separador especificando cada apiario, y la cantidad de alzas de cada tipo a extraer, diciendo al lado la cantidad de miel equivalente de cada colmena. Las cartas de cada colmena seleccionada no puede ser demasiado grandes para que se pueda ver una cantidad razonable de datos y no necesitar hacer tanto scroll.

Otros cambios incluyen:
El gráfico de barras ahora pasará a mostrar de verdad la cantidad de miel extraída de todas las colmenas.

Los apiarios deberían tener otra carta que diga la miel extraida ese año y pasar a decir miel acumulada en la carta ya existente.

Hay que dejar la edición en este caso intacta para editar las cantidades de miel ya que es todo simulado.

Cuando se hace una cosecha, ya sea en masa o individual por nota técnica, se deberá registrar automáticamente como una ganancia en la sección de finanzación.
